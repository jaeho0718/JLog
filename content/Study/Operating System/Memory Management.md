# 메모리 관리

1. **하드웨어 지원** 
	CPU와 메모리간의 접근중 OS가 개입하게되면 성능이 떨어진다. 따라서 메모리 접근 보호기법은 하드웨어가 지원해야한다. Base register와 Limit register를 통해 프로세스에 할당된 여억의 크기를 저장 및 보호한다. (Base register와 Limit register는 Preivileged instruction에 의해 설정 가능)

## 메인메모리
### 주소 할당
메모리 주소 공간에 명령어와 데이터 바인딩은 시점에 따라 구분된다
1. 컴파일 시간 (Compile time): 프로세스가 메모리내에 들어갈 위치를 컴파일 시간에 알 수 있으면 절대코드 생성 가능하다.
2. 적재 시간 (Load time): 컴파일러가 이진코드를 **재배치 가능 코드**로 생성, 메인 메모리로 적재시 심볼과 번지수 바인딩된다.
3. 실행 시간 (Execution time): 프로세스가 실행되는 도중 다른 세그먼트로 이동하게되면, 바인딩이 실행시간에 이뤄진다.
### 논리 주소와 물리적 주소
4. Logical address space
	프로그램에 의해 생성된 모든 논리주소 집합
5. Physical address space
	논리주소와 일치하는 모든 물리 주소 집합을 물리 주소 공간이라한다.

**메모리 관리 장치(MMU)**에 의해 가상 주소(논리적 주소)가 물리 주소로 변환된다.

### 동적
* **동적적재:** 동적 적재에서는 실제 호출되기전까지 메모리가 아닌 재배치 가능 상태로 디스크에서 대기한다. 호출 시 재배치 가능 연결 적재기( Relocatable linking loader)가 불려 요구된 루틴을 메모리로 가져온다.
- **동적 연결 라이브러리(DLL)**: 공유라이브러리라고도 불린다. 라이브러리를 여러 프로세스간에 공유가 가능하므로 DLL인스턴스가 하나만 존재해도된다. (OS 지원이 필요하다.)

### 메모리 할당 방식
가변 파티션 기법: 운영체제는 사용 가능한 메모리 부분과 사용 중인 부분을 나타내는 테이블 유지
	각 파티션은 하나의 프로세스만 적재 가능하다.
-  Hole: 사용 가능한 메모리 블록

#### 연속 메모리 할당
1. **First-Fit**: 첫번째 사용 가능한 가용공간 할당
2. **Best-Fit**: 사용 가능한 공간 중 가장 작은 것 선택
3. **Worst-Fit**: 가장 큰 가용 공간 선택

> [!NOTE] 단편화 (Fragmentation)
> 1. 외부단편화: 프로세스들이 적재되고 제거되는 작업을 반복하다보면 너무 많은 수의 작은 가용 공간들로 단편화된다. 외부 단편화의 경우 **압축(Compaction)**에 의해 해결될 수 있는데 프로세스들의 재배치가 실행시간에 동적으로 일어나는 경우에만 가능하다. 
> 2. 내부단편화: 메모리를 먼저 작은 공간들로 분할하고, 프로세스가 요청하면 분할된 크기의 정수배로 할당한다. 이때 할당된 공간이 요구된 공간보다 클 수 있느데 남은 부분이 내부 단편화다.

#### Paging
- Physical memory는 **Frame**로 같은 크기 블록으로 나눠진다.
- Logical memory는 **Page**로 같은 크기의 블록으로 나눠진다.
- Address = Page number (페이지 테이블 접근시 사용) + Page offset (참조되는 프레임 내에서의 위치)

1. 페이지 번호 P를 추출하여 페이지 테이블 인덱스로 활용
2. 페이지 테이블에서 프레임 넘버 추출
3. 논리 주소의 페이지 번호를 프레임 번호로 바꾼다.

모든 프레임이 프로세스에 할당될 수 있기 때문에 **외부 단편화가 발생하지 않는다.** 그러나 마지막 프레임에 내부 단편화가 발생할 수 있다.

사용자 입장에서는 메모리하나의 연속된 공간으로 보이지만, 실제로는 프레임 단위로 분산되어있다.
- **프레임테이블**: 운영체제가 물리 메모리 관리를 위해 프레임에 대한 정보를 담은 자료구조
- HW 지원: 프로세스별 페이지 테이블을 가지고 있으므로, 프로세스가 선택되면 레지스터를 다시 적재하고 저장된 사용자 페이지 테이블로부터 적절한 하드웨어 페이지 테이블값을 다시 적재해야된다. 대부분의 컴퓨터는 페이지 테이블을 메인메모리에 저장하고 페이지 테이블 기준 레지스터로 하여금 페이지 테이블을 가리키도록 한다.
- **TLB**(Translation Look-Aside Buffer): 페이지 테이블번호를 받아 프레임넘버 반환 (캐시)
