# 핀토스 파일 구조

```bash
pintos/src
├── threads
├── userprog
├── vm
├── filesys
├── devices
├── lib
│   ├── kernel
│   └── user
├── tests
├── examples
├── misc
└── utils
```
- `threads/`: 기본 커널에 대한 소스코드, **프로젝트 1에서 수정할 것이다.**
- `userprog/`: 사용자 프로그램 로더를 위한 소스, **프로젝트2에서 수정할 것이다.**
- `vm/`:  거의 비어있는 폴더다. **프로젝트3에서 가상메모리를 해당 디렉토리에서 구현할 것이다.**
- `filesys/`: 기본 파일시스템에 관한 소스코드, **프로젝트2에서 시작해 프로젝트4까지 수정할 예정이다.**
- `devices/`: I/O디바이스 인터페이스를 위한 소스코드. **프로젝트1에서 `timer`구현을 위해 수정할 것이다.**
- `lib/`: 표준 C라이브러리 부분을 구현. 해당 폴더는 핀토스커널과 사용자 프로그램으로 컴파일된다. (프로젝트2) 커널 코드와 사용자 프로그램에서 해당 폴더의 헤더가 `#include`를 통해 사용될 수있다.
	- `lib/kernel/`: Pintos kernel에만 포함된 c 라이브러리 부분. **커널코드에서 자유롭게 사용해도 되는 일부 데이터 타입이 포함되어있다.** 커널에서 해당 경로가 `#include`로 사용 가능하다.
	- `lib/user/`: 핀토스의 사용자 프로그램에서 사용되는 C라이브러리 중 일부. 사용자 프로그램에서 해당 경로가 `#include`로 사용가능.
- `tests/`: 각 프로젝트에대한 테스트.
- `examples/` : 프로젝트2에서 사용할 사용자 프로그램 예씨
- `misc/` + `utils/`: 핀토스 설치 시 사용.

## Threads 폴더
해당 폴더에서 핀토스 프로그램을 CMake를 통해 빌드를 하면 `build`폴더가 생긴다.
해당 경로에는 다음 주목할만한 파일들이 있다.
1. `Makefile`: `pintos/src`에 있는 `Makefile.build` 복사본. 어떻게 해당 커널을 빌드하는지 설명한다.
2. `kernel.o`: 전체 커널에 대한 오브젝트파일, 디버그 정보를 가지고 있으므로 GDB를 이용해 디버깅할 수 있다.
3. `kernel.bin`: 커널의 메모리 이미지.
4. `loader.bin`: 커널 로더의 메모리 이미지. 디스크에서 메모리로 커널을 보내고 시작하기 위한 코드 포함. 정확하게 512 byte다.
