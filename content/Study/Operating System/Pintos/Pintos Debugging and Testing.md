# 핀토스에서 디버깅과 테스트

## 핀토스 설정
> Bochs에서만 가능하다!!

 컴퓨터 시스템에서 실행할때마다 같은 상황을 재연하기 위해서는 동일한 환경이 보장되어야한다. ==핀토스는 이와 같은 재생산을 기본적으로 제공한다. ==하지만 테스트시 이러한 재생산성을 여러번 테스트하는 것을 무의미하게 만든다.*(테스트는 다른 상황에서 실행해야하는 경우가 있으므로)* 
 1. **랜덤 타이머 인터럽트 설정**
 ```bash
 pintos --qemu -- -q -j seed run ///
```
`-j seed`를 통해 **타이머 인터럽트가 랜덤하게 실행되게 만들 수 있다.** *(Bochs에서만 가능)*

2. 리얼타임
```bash
pintos --qemu -- -q -r run ///
```
`-r`을 통해 실제 실행환경처럼 정확한 시간에 맞게 할 수 있다. *(Bochs에서는 1초가 정확히 1초가 아닌 경우가 있으므로, 정확성을 위해 다음을 설정해줘야한다.)*
> qemu에서는 real-time만 지원.


## 핀토스 테스팅
핀토스에서 각 프로젝트를 만들고 테스트하려면 해당 경로에서 `make check`를 실행하면 된다.
```bash
make check
```

### 시뮬레이터 설정
프로젝트1에서는 `Bochs`가, 나머지는 `Qemu`가 빠르므로 `make check`시 적절한 시뮬레이터를 자동으로 선택해 실행한다.
직접 시뮬레이터를 선택하고 싶으면 아래와 같이하면 된다.
```
make chekc SIMULATOR=--qemu
```

### 개별 테스트 실행
`build`경로에서 `.result`로 된 파일을 실행. 결과는 `t.result`에 출력
```bash
make testse/threads/alarm-multiple.result
```
이미 테스트가 된 상태라면 `up to date`라는 문구가 뜬다. 재실행하고 싶으면 `make clean`을 통해 하자.

### 테스트 진행 사항 보기
```
make check VERBOSE=1
```

### 핀토스 옵션 전달
앞서 다뤘던 핀토스 옵션에 관하여 이를 `make`실행시 전달할 수 있다.
```bash
make check PINTOSOPTS='...'
```
