# 운영체제 구조

## 운영체제에서 제공해야하는 서비스
1. 사용자 인터페이스: GUI, Touch Screen Interface, Command Line Interface ([[Operating System Structures#운영체제가 제공하는 인터페이스]])
2. 프로그램 수행: 프로그램 적재 및 실행, 종료 (비정상 종료도 포함)
3. 입출력 연산: 입출력 수행 수단 제공
4. 파일 시스템 조작: 파일 생성, 삭제, 소유권 확인등 파일 시스템 제공
5. 통신: 프로세스간 정보 교환 방식 제공 (공유 메모리, 메시지 전달등)
6. 오류 탐지: 각 유형의 오류에 대한 적당한 조치 제공
7. 자원 할당: 실행중인 다수 프로세스에 자원 할당 및 관리
8. 기록 작성 (Logging): 프로그램이 사용중인 컴퓨터 자원 추적, 사용통계
9. 보호, 보안: 프로세스 소유권, 시스템 보호 및 보안

## 운영체제가 제공하는 인터페이스
1. **Command Interpreter**: 명령어 해석기 (Shell)
	셀 스크립트: 셸 명령어 모음
	a. 명령어 자체가 실행할 코드를 포함할 경우: 해석기의 크기 증가
	b. 시스템 프로그램으로 처리: 해석기는 메모리에 적재되어 실행할 파일 식별 (`rm file1.txt`)
1. **Graphical User Interface**
2. **Touchscreen Interface**

## 시스템 구성요소와 구성요소간 연결
### 1. System Call
운영체제에 의해 사용 가능하게된 서비스에 대한 인터페이스 제공
- **RTE(실행시간 환경)**: **시스템 콜 인터페이스 제공**, 특정 응용프로그램을 실행하는데 필요한 전체 소프트웨어와 라이브러리 또는 로더와 같은 소프트웨어
- **시스템 콜 인터페이스**: API 함수의 호출을 가로채 운영체제에 필요한 시스템 콜 호출
- 운영체제 매개변수 전달
	- 레지스터를 통해 전달
	- 블록 방법: 매개변수를 메모리 내 블록이나 테이블에 저장, 레지스터에 해당 메모리 주소 저장
	- 스택 방법: 프로그램은 매개변수를 스택에 푸시, 운영체제는 스택의 값을 팝
- **시스템 콜 유형**
	1. **프로세스 제어**: 새로운 프로세스 생성, 프로세스 속성 결정 및 재설정, 프로세스 종료, 프로세스 생성 후 일정 시간만큼 대기 및 특정 이벤트가 일어날 때 까지 대기, 프로세스 잠금(공유 데이터 일관성 제공)
	2. **파일 관리**: 파일 생성, 삭제, 열기, 읽기, 쓰기, 위치 변경, 되감기, 파일 닫기, 파일 및 디렉토리 속성 설정 및 흭득
	3. **장치 관리**: 운영체제에 의해 제어되는 **다양한 자원을 장치로 간주**, 장치 사용 요청 및 방출, 장치 읽기 및 쓰기, 위치 변경 (많은 운영체제가 파일과 장치를 File device structure로 결합)
	4. **정보 유지 관리**:시간, 날짜, 운영체제 정보와 같은 시스템 정보 제공, 프로세스 정보 제공
	5. **통신**: 각 프로세스는 프로세스 이름을 가지고 있으며, 운영체제에 의해 동등한 식별자로 변환
		1. 메시지 전달
		2. 공유 메모리
	6. **보호**

> [!NOTE] Application Programming Interface
> 응용프로그래머가 사용 가능한 함수의 집합 명시, UNIX와 Linux 시스템의 경우 C언어로 제공되는 프로그램을 위해 libc 라이브러리 제공
> ex) Windows API, POSIX API, Java API

### 2. 시스템 서비스
시스템 유틸리티 제공 (프로그램 개발과 실행을 위해 더 편리한 환경 제공)
- 파일 관리
- 상태 정보
- 파일 변경
- 프로그래밍 언어 지원
- 프로그램 적재 및 수행: Absolute loader, Relocatable loader, Linkage Editor, Overlay loader등 제공
	- 링커와 로더
- 통신: 프로세스, 사용자, 컴퓨터 시스템 사이 가상 접속을 위한 기법 제공
- 백그라운드 서비스
	Daemon 서비스: 항상 실행되는 시스템 프로그램 프로세스 (서비스, 서브시스템)

> [!NOTE] 시스템 부트
> **부팅**: 커널을 적재하여 컴퓨터 시작하는 과정
> 1. 부트스트랩 프로그램 혹은 부트로더가 커널위치 탐색
> 2. 커널 메모리에 적재 및 실행
> 3. 커널이 하드웨어 초기화
> 4. 루트 파일 시스템 마운트
> 
> **다단계 부팅**: 
> 1. BIOS(소형 부트로더) 실행
> 2. 부트 블럭에 위치한 두번째 부트로더 적재 (소형 부트로더의 임무)
> 3. 전체 운영체제를 메모리에 적재 및 실행
> 
> **UEFI (Unified Extensible Firmware Interface)**: 하나의 부팅 관리자, 다단계 BIOS 부팅 과정보다 빠르다.
> **GRUB**: 공개 소스 부트 스트랩 프로그램, 부팅 시 커널 매개변수 수정 및 부팅 가능한 커널 선택 가능
>
